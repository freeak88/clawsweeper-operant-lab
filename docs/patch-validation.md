# Patch Proposal Validation

Patch Proposal Validation evaluates `patch-proposal.json` artifacts before any future shadow execution or supervised implementation step.

It is artifact-only. It does not apply patches, modify source files from the proposal, create commits, push branches, create PRs, mutate GitHub, dispatch repairs, or change scheduler/apply/automerge behavior.

## Input

```text
patch-proposal.json
```

The input is generated by Supervised Patch Generation.

## Output

```text
patch-validation.json
patch-validation.md
```

The JSON output has this shape:

```json
{
  "validation_id": "...",
  "patch_id": "...",
  "status": "valid",
  "checks": [],
  "blocking_risks": [],
  "warnings": [],
  "recommended_next_step": "eligible_for_shadow_execution",
  "summary": "Patch proposal passed artifact-only validation."
}
```

## CLI Usage

```bash
pnpm run patch-validation -- \
  --patch results/patch-generation/improve-memory/patch-proposal.json \
  --output-root results/patch-validation/improve-memory
```

## Validation Checks

The validator checks:

- patch proposal status is `patch_proposed`
- required fields exist
- `files_to_modify` and `files_to_add` are arrays
- `tests_to_run` is non-empty
- `rollback_plan` is non-empty
- `safety_constraints` is non-empty
- `non_goals` include no commit, no push, no PR creation, no merge, and no GitHub mutation
- proposed files avoid sensitive paths

Missing tests, rollback, safety constraints, or required non-goals block validation.

## Sensitive Paths

Sensitive paths produce `needs_review`, not `valid`:

- `.github/workflows/*`
- `src/clawsweeper.ts`
- `src/repair/*`
- `src/guarded-execution/*`
- `src/policy-dsl/*`
- `package.json`
- `pnpm-lock.yaml`

This keeps high-impact surfaces visible to humans without pretending a proposal is safe.

## Safety Boundary

Patch Proposal Validation does not:

- apply patches
- modify source files from the patch proposal
- create commits
- push branches
- create PRs
- merge PRs
- mutate GitHub
- dispatch repairs
- change scheduler/apply/automerge behavior

## Future Path

The next safe step is shadow patch execution: simulate whether the proposed file changes would fit the approved plan and test requirements without writing the patch to the working tree.
